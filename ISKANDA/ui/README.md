## What is this UI?

This is the **React chat interface** for **Code Lens** — Phase 1 of Engineer AI. It allows you to:

1. **Ingest** any codebase by entering a folder or file path
2. **Chat** with Code Lens about that codebase in plain English
3. **View history** of previously ingested codebases and re-activate them instantly
4. **Reset** and switch to a different codebase anytime

It talks to the Code Lens backend (Express server on port 3001) via REST API calls.

```
You type a question
        ↓
React UI (localhost:5173)
        ↓  POST /ask
Express Backend (localhost:3001)
        ↓
Code Lens Agent → ChromaDB → Groq LLM
        ↓
Answer comes back
        ↓
Renders as formatted markdown in chat
```

## How Each File Works

### `App.tsx`
The **entire UI** lives in this one file. It has:

**State variables:**
```typescript
messages      → array of all chat messages (user + Code Lens)
question      → what user is currently typing
folderPath    → codebase path input value
isReady       → has a codebase been ingested?
isLoading     → is Code Lens currently thinking?
isIngesting   → is ingestion in progress?
ingestedPaths → list of previously ingested paths (from backend)
showHistory   → toggle for history table visibility
```

**Key functions:**
```typescript
checkHealth()        → on load, checks if backend already has data (auto-restore)
loadPaths()          → fetches previously ingested paths from GET /paths
handleIngest()       → calls POST /ingest with folderPath
handleAsk()          → calls POST /ask with question
handleReset()        → calls POST /reset, clears messages
handleKeyDown()      → sends message on Enter key (Shift+Enter = new line)
formatDate()         → formats ISO date to readable Indian locale format
```

**UI sections:**
```
Header         → Engineer AI title + Code Lens phase label + ready/not-ready badge + Reset button
Ingest bar     → folder path input + Ingest button (shown before ingestion)
History table  → previously ingested paths with ⚡ Use buttons (collapsible)
Messages       → scrollable chat history with markdown rendering
Input bar      → question textarea + send button
```

### `App.css`
Full dark theme styling. Key design decisions:
- Background: `#0a0a0f` — very dark, easy on eyes for developers
- Accent: `#a78bfa` — purple, matches the cosmic/Shiva theme
- User messages: right-aligned, dark purple bubble
- Code Lens messages: left-aligned, dark card with border
- Typing animation: three bouncing dots while waiting for response
- Markdown styles: headers, code blocks, bold, lists — all themed
- `word-break: break-all` on code — fixes long file paths overflowing
- Custom slim scrollbar with purple hover effect
- `scroll-behavior: smooth` for natural chat feel

### `main.tsx`
Auto-generated by Vite. Mounts the React app into `index.html`. Never needs editing.

### `vite.config.ts`
Auto-generated. Tells Vite this is a React + TypeScript project. Never needs editing.

## How the UI Connects to Backend

The UI talks to the backend via 4 API calls:

### 1. Health check — on every page load
```typescript
const res = await axios.get("http://localhost:3001/health");
// If isReady = true → Code Lens auto-restores, no ingestion needed
```

### 2. Load paths — on every page load
```typescript
const res = await axios.get("http://localhost:3001/paths");
// Shows previously ingested codebases in history table
```

### 3. Ingest — when user clicks ⚡ Ingest
```typescript
await axios.post("http://localhost:3001/ingest", {
  folderPath: "C:/Projects/my-app/src"
});
// Backend reads + embeds the codebase
// Can take minutes for large projects
```

### 4. Ask — when user sends a message
```typescript
const response = await axios.post("http://localhost:3001/ask", {
  question: "How does authentication work?"
});
console.log(response.data.answer); // Code Lens explanation
```

### 5. Reset — when user clicks 🔄 Reset
```typescript
await axios.post("http://localhost:3001/reset");
// Clears ChromaDB, UI resets to initial state
```

## UI Features

### ✅ Auto-restore on page load
On every page load, the UI checks `/health`. If Code Lens already has data in ChromaDB, it marks itself as ready — no need to re-ingest.

### ✅ Ingestion History Table
Previously ingested paths are shown in a collapsible table with:
- Path (highlighted in blue monospace)
- Date + time ingested
- ⚡ Use button — click to re-activate instantly

### ✅ Chat Interface
- Full scrollable message history
- User messages on the right (purple)
- Code Lens messages on the left (dark card)
- Auto-scroll to latest message

### ✅ Markdown Rendering
Code Lens answers come back as rich markdown:
```markdown
### Authentication Flow
The login is handled in `src/auth/login.ts`...

**Step 1:** User enters credentials
**Step 2:** Token is generated via JWT...
```
Renders with headers, bold text, inline code, and code blocks.

### ✅ File Path Overflow Fix
Long file paths and code in responses no longer overflow the chat bubble — `word-break: break-all` applied to inline code, horizontal scroll on code blocks.

### ✅ Typing Animation
While Code Lens is thinking — three bouncing purple dots show in the chat bubble.

### ✅ Status Badge
Top right shows:
- `● Ready` (green) — codebase ingested, ready to answer
- `● Not ingested` (grey) — needs ingestion first

### ✅ Disabled States
- Input disabled until codebase is ingested
- Buttons disabled during loading/ingesting
- Visual opacity change shows disabled state clearly

### ✅ Keyboard Shortcut
- `Enter` → sends message
- `Shift + Enter` → new line in textarea

### ✅ Reset Button
Appears after ingestion. Clears everything for a fresh codebase.

### ✅ Smooth Custom Scrollbar
Slim purple-accented scrollbar in the messages area. Smooth scroll behavior.

## Issues Faced & Solutions

### 1. `npm run dev` not found — wrong directory
**Error:** `Missing script: "dev"`
**Cause:** Running from root instead of `ui/` folder
**Solution:**
```bash
cd ui
npm run dev
```

### 2. CORS error when calling backend
**Error:** `Access to XMLHttpRequest blocked by CORS policy`
**Solution:** Added `cors()` middleware in `server.ts`:
```typescript
import cors from "cors";
app.use(cors());
```

### 3. Markdown showing as raw text
**Problem:** Code Lens answers showed `##` and `**` instead of formatted text
**Solution:** Install and use `react-markdown`:
```bash
npm install react-markdown
```
```tsx
import ReactMarkdown from "react-markdown";
<ReactMarkdown>{msg.content}</ReactMarkdown>
```

### 4. Messages not auto-scrolling
**Problem:** New messages appeared but user had to manually scroll
**Solution:** Used `useRef` + `useEffect`:
```typescript
const bottomRef = useRef<HTMLDivElement>(null);
useEffect(() => {
  bottomRef.current?.scrollIntoView({ behavior: "smooth" });
}, [messages]);
```

### 5. Enter key submitting with newline
**Problem:** Pressing Enter added a newline instead of sending
**Solution:** `handleKeyDown` prevents default on Enter only:
```typescript
if (e.key === "Enter" && !e.shiftKey) {
  e.preventDefault();
  handleAsk();
}
```

### 6. Loading bubble replaced incorrectly
**Problem:** While new answer loaded, old content flickered
**Solution:** Replace only the last message (loading bubble):
```typescript
setMessages(prev => [...prev.slice(0, -1), { role: "codelens", content: answer }]);
```

### 7. Long file paths overflowing chat bubble
**Problem:** File paths like `src\components\modal\Modal.jsx` broke out of the bubble
**Solution:** Added CSS to `App.css`:
```css
.bubble code {
  word-break: break-all;
}
.bubble pre {
  overflow-x: auto;
}
```

### 8. UI loses state when laptop sleeps / server restarts
**Problem:** After restart, user had to re-ingest entire codebase
**Solution:**
- Backend saves ingested paths to `ingested-paths.json` via `pathStore.ts`
- On server start, auto-restores last active path if ChromaDB has data
- UI calls `/health` on load — if `isReady: true`, skips ingestion screen
